Bridge

A Spanish speaker sends text in Spanish.

An English speaker recieves message translated to English.

Screenshot of the Bridge Android app.

By using Android Studios, Azure, Bing-Translator API, and Java, a team of four built an Adroid SMS app that translated text and voice-recorded messages to a user's dafault language. For example, a Spanish-speaking person conversating with an English-speaking will recieve their messages in Spanish and send messages in Spanish, but the English speaker will recieve the English-translated message.

There were many translation apps before ours, however the team focused a simple, clean interface that would look like any other messenger app. Most translation apps had an input field for typing in a message, and another field that showed the message after translation. The chat interface would compose of both the original and translated message. It looked bulky, so the goal of the Bridge app was to simplify the design.

Only having taken an introductory course in Java, the team faced many challenges trying to complete a demo for UCIHack. Every step of teh project was an obstacle. The first challenge was enabling the app to use the internet so that we were able to create an app that would send SMS messages to another phone. The second challenge was connecting the Bing Translation API so that the SMS messages would be translated. Lastly, which the team could not complete in time, was make a chat interface so that users can chatt directly using the app rather than have a translation app.

Overall, builidng Bridge helped me get comfortable with Android Studdios and be familiar with what APIs are.
